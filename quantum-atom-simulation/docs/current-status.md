# Current Status

## 项目当前定位

这个项目当前应被表述为：

- 一个可构建、可运行、可打包的桌面原子可视化与教学验证平台
- 一个围绕氢样体系、中心场近似、局部谱线对照和 3D 概率云展示构建的科学软件 MVP
- 一个为后续接入更高保真度原子结构、谱学修正和时间依赖模块预留了架构接口的工程基础

它**不是**：

- 研究级高精度原子结构软件
- 已实现 HF / DFT / CI / MCSCF 的多电子量子化学平台
- 已实现真实 TDSE 动力学的量子动画系统

## 当前物理特性

### 1. 严格氢样解析层

当前最严谨、最可靠的物理链路是严格氢样体系：

- `E_n ~ -Z^2 / n^2`
- `r_n ~ n^2 / Z`
- 跃迁能量、频率、波长
- 电偶极选择规则过滤
- 约化质量修正
- 与本地参考谱线做样例对照

适用范围：

- 氢原子
- 氢样离子，例如 `He+`
- 教学中的单电子中心势验证

### 2. 多电子教学近似层

当前多电子链路基于：

- Aufbau 填充
- Pauli 不相容
- Hund 规则
- Slater 屏蔽
- 用 `Z_eff` 替代 `Z` 的氢样替代估算

这层适合：

- 电子组态教学
- 元素趋势讲解
- “为什么多电子原子不再严格氢样”的概念展示

这层不适合：

- 高精度能级预测
- 复杂原子精确谱学
- 研究级多电子结构结论

### 3. 数值求解层

当前已实现径向 Schrödinger 方程的有限差分本征值求解。

它可用于：

- 氢样中心势能级验证
- 收敛曲线展示
- 径向波函数和分布的数值检查

它目前仍然是：

- 单电子
- 中心势
- 径向问题

不是完整三维多体求解。

### 4. Tier 1 中心场层

当前已有一个现象学屏蔽中心势层，可在氢样库仑势之外展示更一般的单电子有效势图像。

这层的价值是：

- 为后续平均场方法预留落点
- 改善“氢样之外全无过渡层”的架构断层

但它仍然不是：

- SCF
- Hartree
- Hartree-Fock

### 5. Tier 3 谱学修正层

当前已接入教学型谱学修正：

- 氢样精细结构
- Zeeman
- 受限的一阶 Stark

它们当前的价值主要是：

- 展示“基线能级/谱线”与“修正后值”的差别
- 让图表和方法标签更接近真实谱学模块结构

但它们仍然不是完整复杂原子谱学程序。

## 当前技术特性

### 1. 图形与界面

当前 GUI 和渲染主线是：

- OpenGL 3.3 Core
- GLFW
- GLEW
- ImGui Docking
- 统一 Scene / Inspector / Physics / Plots / Performance / Help / Log 多面板工具界面

场景支持：

- 轨道
- 概率点云
- 体切片
- 相位着色
- 自动演示脚本
- 适配全部 / 仅核 / 仅轨道 / 仅云

### 2. 图表与交互

图表当前支持：

- 鼠标悬停提示
- 点击选中
- 框选缩放
- 平移
- 重置视图
- 局部窗口数据联动
- 当前窗口 CSV 导出

### 3. 工程与交付

当前工程已具备：

- `xmake` 主构建链
- `build / run / package / smoke test` 脚本
- `dist/` 打包发布目录
- 本地测试与冒烟启动验证
- Windows CI 骨架

### 4. 数据与可追溯

当前结果对象已逐步带上：

- 方法标签
- 数据来源
- provenance
- validation record

元素静态元数据已覆盖：

- 完整周期表 `Z=1..118`
- 中文名
- 基本质量信息

## 当前不足

### 物理不足

- 多电子仍是 `Z_eff + Slater` 教学近似
- 没有电子关联
- 没有严格反对称多电子波函数
- 没有 HF / DFT / CI / MCSCF
- 没有 Lamb shift / QED / hyperfine
- 没有真实时间依赖 TDSE 动力学

### 数据不足

- 参考谱线仍以本地示例资产为主
- 还没有完整外部数据库缓存/映射层
- 元数据完整不代表理论精度完整

### 技术不足

- 当前优先验证平台仍是 Windows x64
- Linux/macOS 还缺同等级别的实测交付闭环
- 图表系统是项目内自绘方案，不是完整通用 plotting framework
- GPU 侧采样和更高阶 LOD 还有继续优化空间

### 产品化不足

- 当前仍更偏开发态 MVP，而不是正式发布版科学软件
- UI 虽然已经可用，但还可以继续做一致性、信息密度与引导优化
- 仍需持续清理历史文案和局部老逻辑

## 当前最值得继续推进的方向

1. 数据层升级：做可缓存的外部数据库映射。
2. Tier 1.5 / Tier 2：做更可靠的有效势或小体系 SCF 原型。
3. 图表与报告升级：导出、筛选、报告模板继续自动化。
4. 可视化继续优化，但始终不能越过物理真实性边界。
